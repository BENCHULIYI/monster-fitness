## 第十二轮：器材切换功能修复 + 底部导航栏适配

### 用户问题：
1. 计划栏目内器材依然不能切换
2. 底部导航栏不能完全落底，看起来很奇怪

### 问题根源1（器材切换）：
- `selectEquipInModal()` 函数中，点击器材卡片时会重新调用 `showEquipModal()`
- 但这个函数每次都会把 `modalSelections` 重置为当前状态（`state.equipment`）
- 所以用户的选择被立即覆盖了！

### 修复方案1：
1. **分离初始化和刷新逻辑** - 只在弹窗首次打开时从 `state` 初始化选择
2. **新增 `renderEquipGrid()` 函数** - 单独负责渲染器材网格
3. **简化点击处理** - 点击器材卡片时只更新选择并重新渲染网格，不重置
4. **新增 `selectEquipInModal()` 函数** - 专门处理器材选择，调用 `renderEquipGrid()` 刷新UI

### 问题根源2（底部导航栏）：
- `.bottom-nav` 使用默认流式布局，没有固定定位
- 在某些手机上，底部安全区域（safe-area-inset-bottom）没有被正确计算
- 页面内容区域的 `padding-bottom` 不够，导致内容被导航栏遮挡

### 修复方案2：
1. **导航栏改为固定定位** - `position: fixed; bottom: 0`，始终贴在屏幕最底部
2. **增强安全区域适配** - 使用 `env(safe-area-inset-bottom, 0px)` 获取iPhone底部安全区域高度
3. **页面内容区域调整** - 给 `.page` 添加足够的 `padding-bottom`，避免内容被固定导航栏遮挡
4. **app-container高度调整** - 从 `height: 100%` 改为 `min-height: 100%`，适应内容高度
